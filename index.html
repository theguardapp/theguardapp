System health

A dashboard built for clarity

Live battery, storage, CPU, and thermal metrics anchor a single readable view, with hardware-grade memory diagnostics one tap away showing used, active, and kernel allocation in real time.

  • Battery, storage, and CPU temp at a glance
  • Memory bandwidth, zRAM, and thrashing health
  • Root state and SELinux mode surfaced honestly
Storage intelligence

Storage that explains itself

More than a "free space" bar: detect the underlying flash interface (UFS / eMMC), break space down by apps and reclaimable cache, and verify encryption type and mountpoint at a glance.

  • UFS / eMMC interface detection
  • App-by-app space breakdown with reclaimable cache
  • Encryption mode (FBE / metadata) and mountpoint visibility
Battery & charging

Battery health, in forensic detail

Track capacity degradation against the original hardware design, count every charge cycle, and watch live charging analytics surface the actual protocol your charger is negotiating at the millisecond.

  • State of Health: capacity vs. original design
  • Charging protocol detection (USB / PD / QC)
  • Live wattage, voltage, current, and charge phase
Thermal & power

Heat tracked, throttling caught

Multi-tier thermal alerts and throttling detection are paired with a per-component power-draw breakdown, so you know exactly which radio, sensor, or core is costing you the most milliamps.

  • CPU and battery temperature trend graphs
  • Top power consumers ranked by mA & share
  • Kernel forensics with anomaly confidence score
Forensic recovery

Recovery without the cloud

Signature-based file carving over internal storage and OTG media, with a software write-blocker, live drive-health monitoring, and cryptographically signed reports.

  • Software write-blocker enforced on every OTG mount
  • Selective re-carving and SMART-style drive health
  • Signed forensic reports with full chain-of-custody data
Privacy & app audit

  • Activity, foreground time, WiFi & mobile data per app
  • Identity, provenance, and packaging architecture

Questions & answers

Everything you need to know

The straight answers to the questions we get asked most often. If something's missing, drop us a line.

No. The app has no backend, no analytics SDK, no crash-reporting service, and no telemetry of any kind. You can verify this by inspecting your device's outbound network traffic while the app is running; there are no calls to our infrastructure, because we don't operate any.
Never. The Guard works fully offline. There is no sign-up, no email gate, no "unlock with login" flow. Install the app and start using every feature immediately.
Each module needs its own permissions to function: storage access for forensic recovery, USB host for OTG analysis, usage stats for performance metrics, and notifications for thermal and security alerts. We explain each permission in-app before requesting it, and you can revoke any of them via Android settings at any time.
It's a robust software-level safeguard that blocks every write-direction SCSI command on the OTG bus. For evidentiary work where chain of custody is critical, we recommend pairing it with a hardware write-blocker. The Guard's reports are cryptographically signed regardless.
The Guard supports Android 8.0 (Oreo) and later. A handful of advanced forensic features, selective re-carving and drive-health stats over OTG, require Android 10+ for the underlying USB and storage APIs.
The app is free, and every feature you see is included. We have no plans to introduce a paid tier. If anything ever changes, the existing feature set will remain free for current users.
Open any module, tap the share icon, and choose either PDF or signed-text export. By default the share chooser hides cloud-storage targets so local-only sharing is the obvious path; you can still pick any installed app explicitly.
